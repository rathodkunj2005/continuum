# Continuum iOS / watchOS workspace

## What is in this folder

- `Continuum.xcodeproj`: iOS + watchOS project generated by XcodeGen.
- `project.yml`: source-of-truth XcodeGen spec.
- `Continuum/`: iOS SwiftUI app shell with Ask, Memories, Capture, Status, Settings,
  Pairing, and App Intents scaffolding.
- `Continuum Watch/`: watchOS SwiftUI scaffold with Ask, Remember, Recent, Status screens.
- `ContinuumKit/`: shared Swift package (DTOs, KeychainStore, CompanionClient,
  PairingFlow, ConnectionStatusService, offline capture queue, watch bridge,
  and runnable check suite.

## Regenerate project

From repo root:

```bash
xcodegen generate --spec apps/ios/project.yml --project apps/ios
```

## Full Xcode prerequisite

`xcodebuild` and simulator validation require the full Xcode app, not only
Command Line Tools.

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license accept
xcodebuild -version
xcrun simctl list devices
```

## Real-device install

This repo can install a development build directly to a trusted iPhone. The
phone app embeds the watchOS app; use `CONTINUUM_WATCH_DEVICE_ID` only if you want to
install the watch target directly as a separate step.

1. Connect and trust the iPhone from Finder/Xcode.
2. Find the device id:

```bash
xcrun devicectl list devices
```

3. Install a signed development build:

```bash
CONTINUUM_IOS_TEAM_ID=<apple-team-id> \
CONTINUUM_IOS_BUNDLE_PREFIX=com.<your-name>.continuum \
CONTINUUM_IOS_DEVICE_ID=<iphone-device-id> \
scripts/ios/install-real-device.sh
```

4. Start the Mac runtime (`npm run tauri dev`), open Continuum Settings -> Paired
   devices, generate a pairing QR code, then scan it from the iPhone Pair view.

The iPhone target includes local-network and camera usage strings because real
devices need both: local-network access for the Mac companion API and camera
access for QR pairing.

## Local verification (requires full Xcode)

Build for simulator:

```bash
xcodebuild -project apps/ios/Continuum.xcodeproj \
  -scheme Continuum \
  -destination 'platform=iOS Simulator,name=iPhone 15' \
  build
```

Build watch app scheme:

```bash
xcodebuild -project apps/ios/Continuum.xcodeproj \
  -scheme "Continuum Watch" \
  -destination 'platform=watchOS Simulator,name=Apple Watch Series 9 (45mm)' \
  build
```

Run tests for the package (Xcode environment):

```bash
cd apps/ios/ContinuumKit
swift test
```

Run tests in CLI-only environments (no XCTest dependency):

```bash
cd apps/ios/ContinuumKit
swift run ContinuumKitCheck
```

## iPhone smoke (simulator)

1. Start Continuum desktop runtime on Mac (`npm run tauri dev`).
2. Open desktop Settings -> Paired devices.
3. Generate pairing code/QR.
4. Run iOS app on simulator from `Continuum.xcodeproj`.
5. Open Pair screen and paste scanned QR payload JSON (temporary draft UX).
6. Tap **Complete pairing**.
7. Go to Ask tab and run a query.
8. Go to Memories tab and run a search + open detail.
9. Go to Capture tab and submit a note, then disable network and submit another
   note to verify offline queue fallback.
10. Go to Status tab and tap **Refresh status**.
11. Verify reachability and capture/model/storage fields are populated.
12. Tap **Pause capture**, then **Resume capture**, and verify status updates.

## Watch smoke (simulator / device)

1. Build and run `Continuum Watch` scheme.
2. Validate Ask, Remember, Recent, and Status screens load.
3. Confirm iPhone bridge handshake path (currently draft transport wiring).
4. On real device, document quick-action pause and remember behavior.

## Known environment blocker encountered in Codex run

The current Codex host had only Command Line Tools active
(`/Library/Developer/CommandLineTools`), so these commands were unavailable:

- `xcodebuild -version` (requires full Xcode app/toolchain)
- `xcrun simctl list devices` (simulator tools missing)

Simulator/device smoke validation must be run on a machine with full Xcode
installed and selected (`sudo xcode-select -s /Applications/Xcode.app/Contents/Developer`).
